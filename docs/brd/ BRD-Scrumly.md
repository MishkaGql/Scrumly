## 1. 🧭 Общее описание проекта

**Scrumly** — система управления проектами и отслеживания задач, поддерживающая методологии **Agile (Scrum)** и **Kanban**, с интеграцией к сервисам, необходимым для разработки ПО.


## 2. 📖 Предыстория и суть проекта

Проект является современной альтернативой **Jira**.

Jira, несмотря на богатый функционал и высокую интеграцию с другими сервисами, сегодня имеет ряд критических проблем:

- 🐢 Устаревший интерфейс с медленными ответами на запросы
- 🔔 Бесполезные уведомления на почту при каждом действии
- 🗑️ Множество устаревших, давно не используемых фич
- 😵 Запутанный UX, требующий долгого изучения

**Scrumly** предлагает:

- Удобный, не перегруженный, но функционально богатый интерфейс
- Только действительно нужные функции
- Максимально простую интеграцию с **GitHub / GitLab**
- Реферальную систему, открывающую расширенный функционал
- Понятную документацию по экосистеме, доступную каждому пользователю

## 6. 🗺️ Границы проекта

### ✅ В границах проекта

- Все компании, предоставляющие услуги в интернете: **сайты, мобильные приложения, игры**
- Команды разработчиков: **малые, средние, крупные**
- Команды энтузиастов

### ❌ Вне границ проекта

- Компании, не работающие в сети интернет
- Команды из **2–3 человек** с небольшим SPA без планов на масштабирование

### 🎁 Ожидаемые результаты

**Внешние:**

- Сайт и мобильное приложение для совместной разработки внутри компаний и команд
- Интеграция с GitHub / GitLab, платёжными системами (Visa, Mastercard), реферальной системой

**Внутренние:**

- Высокие амбиции при возможности появления более сильных аналогов за время разработки
- Баланс между простотой и функциональностью — разные пользователи имеют разные ожидания

## 8. ⚙️ Функциональные требования

### 🔐 Аутентификация и регистрация

Система должна поддерживать регистрацию и вход через:

- Ручной ввод данных (email + пароль)
- **Google** OAuth
- Номер телефона
- **Яндекс** OAuth
- Интеграция с **Keycloak** _(планируется)_

---

### 📁 Управление проектами

Система должна позволять пользователю:

- Создать проект **в 1 клик** с готовым шаблоном (**Scrum** или **Kanban**)
- Детально настроить проект вручную: описание, видимость, роли, разрешения
- Переключаться между шаблонами **с сохранением** всех настроек задач и статусов

---

### 📝 Управление задачами

Система должна позволять:

- Создавать **Epic** и **User Story** в один клик
- Настраивать кастомные правила создания задач: роли, разрешения, ограничения

---

### 👤 Роли и права

|Роль|Уровень доступа|
|---|---|
|**Пользователь**|Базовый доступ к платформе|
|**Участник проекта**|Работа с задачами внутри проекта|
|**Модератор**|Временный бан, временное назначение ролей и прав|
|**Администратор**|Полные права: изменение UI, управление ролями, назначение прав другим участникам|
