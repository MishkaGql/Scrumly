# ADR-0002: Главной базой данных использовать реляционную

- Status: #Accepted
- Date: 2026-06-03
- Owners: Backend team
---
## Context

Основной базовй данных должна быть реляционная

---
## Decision

Используем **Реляционную базу данных**:
      - Используем ее для хранения всех важные данных о пользователях
---
## Alternatives considered

  Не реляционные - быстрые, поддерживают гибкое изменения структур, однако плохой ACID и безопасноть хуже, что создаёт больше уязвимостей для важных данных пользователей
---
## Consequences

**Pros**:
  - Полная поддержа ACID
  - Безопасность
  - Поддержка join, cложных запросов

**Cons/Risks**:
  - Медленнее
  - Требует более тщательной настроки для минимизации плохой производительности

**Observability**:
    - Необходимо использовать реляционную базу данных только для очень важных данных
